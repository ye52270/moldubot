# 역할 Taxonomy 설정 가이드

이 문서는 몰두봇의 "인물 역할표" 규칙을 코드 수정 없이 운영 중에 조정하는 방법을 설명합니다.

## 1. 무엇이 바뀌었나
- 기존: 역할 키워드 규칙이 코드에 하드코딩됨
- 변경: 규칙을 `config/role_taxonomy.json`에서 읽음
- 동작: 파일 수정 후 저장하면 다음 요청부터 자동 반영됨(재시작 불필요)

## 2. 설정 파일 위치
- 기본 경로: `config/role_taxonomy.json`
- 커스텀 경로: 환경변수 `ROLE_TAXONOMY_CONFIG_PATH`로 지정 가능

## 3. 파일 형식
```json
{
  "version": "1.0",
  "default_roles": {
    "to": "수신/실행 대상",
    "cc": "공유 대상",
    "unknown": "역할 미상"
  },
  "role_hints": [
    {"keyword": "요청", "role": "요청자"},
    {"keyword": "검토", "role": "검토 담당"},
    {"keyword": "조치", "role": "실행 담당"}
  ]
}
```

## 4. 운영자가 자주 하는 변경
- 새 키워드 추가: `role_hints`에 항목 추가
- 역할 명칭 변경: 기존 항목의 `role` 값 수정
- 미매핑 기본값 변경: `default_roles.unknown` 수정

## 5. 반영 확인 방법
1. 설정 파일 저장
2. 챗봇에 다음과 같이 입력
- `현재메일 본문에 명시된 사람들의 역할을 표 형태로 정리해줘`
3. 역할표에서 수정한 역할명이 반영되는지 확인

## 6. 주의사항
- JSON 문법 오류가 있으면 기본 규칙으로 자동 fallback 됩니다.
- 키워드는 위에서 아래 순서대로 매칭되므로, 더 구체적인 키워드를 위에 두는 것을 권장합니다.
- `role_hints`가 비어 있으면 기본 규칙 세트가 사용됩니다.
